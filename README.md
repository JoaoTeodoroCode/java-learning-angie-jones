# 🚀 Java for Test Automation - Learning Journey [COMPLETE]

Este repositório documenta a minha jornada completa e conclusão do curso de **Java** da **Angie Jones** na **Test Automation University (TAU)**. O foco foi construir uma base sólida de programação para automação de testes profissional.

## 📌 Sobre o Projeto
Este repositório é o resultado de um estudo prático focado em dominar a **Programação Orientada a Objetos (POO)**. O objetivo foi aprender a estruturar código que não apenas funciona, mas que é escalável e fácil de manter em ambientes de automação de testes.

---

## 🧱 Fundamentos Críticos (A Base de Tudo)
Antes de avançar para a arquitetura, consolidei a sintaxe essencial que permite ao código tomar decisões e processar dados:

* **Tipagem e Variáveis:** Manipulação de dados (`String`, `int`, `double`, `boolean`) com foco em boas práticas.
* **Estruturas de Controlo:** Domínio de fluxos condicionais (`if/else`, `switch`) e estruturas de repetição (`for`, `while`) para controlo de execução.
* **Modularização:** Criação de métodos, entendimento de parâmetros e retorno de dados para evitar repetição de código.

## 🧠 Conceitos de Engenharia Aplicados (POO)
Nesta fase, o foco mudou para a organização e proteção do código através da Orientação a Objetos:

* **Encapsulamento:** Uso de modificadores `private` e métodos `getters/setters` para proteção de dados sensíveis.
* **Herança e Polimorfismo:** Construção de hierarquias de classes e comportamentos dinâmicos, fundamentais para frameworks de teste.
* **Abstração:** Uso de interfaces e classes abstratas para definir contratos de comportamento entre objetos.

## 🛠️ Gestão de Dados e Resiliência (Avançado)
* **Collections Framework:** Uso de `List`, `Set` e `Map` para gerir múltiplos elementos (essencial para capturar listas de elementos numa página web).
* **Tratamento de Exceções:** Implementação de `try-catch-finally` e propagação de erros com `throws`, garantindo que os testes sejam resilientes a falhas inesperadas.

---

## 🏆 Destaque Final: Coin Toss Game (Projeto de Conclusão)
O projeto final consolida todos os pilares acima num simulador de jogo de "Cara ou Coroa".

### O que foi implementado:
* **Lógica Aleatória:** Uso da classe `java.util.Random` para simular o lançamento da moeda.
* **Separação de Responsabilidades:** Classes distintas para a Moeda (`Coin`), Jogador (`Player`) e o fluxo do Jogo (`CoinTossGame`).
* **Imutabilidade:** Uso de constantes (`static final`) para definir os estados fixos do jogo.
